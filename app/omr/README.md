# OMRChecker Integration Guide

This directory contains an integrated version of the [OMRChecker](https://github.com/Udayraj123/OMRChecker) project, used for automatic grading and processing of bubble sheets. We use the dev branch of OMRChecker, customized to meet our system requirements.

## Directory Structure

```
omr/
├── OMRChecker/         # OMRChecker core code
├── pdf_to_images.py    # PDF to image conversion and preprocessing
├── process_omr.py      # OMR processing main program
└── README.md           # This documentation
```

## Main Features

1. **PDF Processing**: Convert PDF answer sheets to images
2. **Image Preprocessing**: Detect, correct, and standardize images
3. **OMR Grading**: Recognize and grade answer sheets
4. **Result Generation**: Output grading results in JSON format

## Usage Process

### 1. Install Dependencies

First, install the required dependencies:

```bash
pip install -r omr/OMRChecker/requirements.txt
pip install pdf2image pyzbar opencv-python numpy pillow
```

### 2. PDF to Image Conversion

Use `pdf_to_images.py` to convert PDF answer sheets to images:

```python
from omr.pdf_to_images import pdf_to_images

# Single-sided answer sheets
pdf_to_images("input_dir", "output_dir")

# Double-sided answer sheets
pdf_to_images("input_dir", "output_dir", double_pages=True)
```

### 3. OMR Processing

Use `process_omr.py` to process images and grade them:

```python
from omr.process_omr import process_omr

# Process and grade
results = process_omr(
    template_file="/path/to/template.json", 
    image_dir="/path/to/images",
    answer_key={"1": "A", "2": "B", ...}
)
```

## System Integration

This OMR module integrates with the following parts of the system:

1. **Template Generation**: Templates generated by `examController.js` are used by the OMR processing program
2. **Result Processing**: OMR results are processed by the `saveResults` API and stored in the database
3. **Image Processing Pipeline**: The service at the `/split_pdf` endpoint calls the OMR module

## Custom Bubble Sheet Recognition

The system uses QR code markers to ensure precise recognition:

1. Each page has QR codes in all four corners
2. The first page uses QR codes numbered 1-4
3. The second page uses QR codes numbered 5-8
4. The system can detect page orientation and automatically correct it

## Important Notes

1. OMRChecker works best with scans of sufficient contrast (300 dpi recommended)
2. The system supports PDF files as input and processes them automatically
3. The template file defines the exact positions of bubbles on the answer sheet
4. The system is configured to process at 150 ppi, ensuring all distances are integer pixels

## Troubleshooting

Common issues:

1. **Inaccurate bubble recognition**: Check scan quality and contrast
2. **Page recognition errors**: Ensure QR codes are clearly visible
3. **Slow processing**: Reduce batch size or improve processing machine configuration
